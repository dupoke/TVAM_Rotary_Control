# 旋转台控制程序总体实施计划（Master Plan）

> 依据：`CONTROL_SOFTWARE_TECH_SPEC.md`（V1.1）
> 执行原则：最小改动、单步闭环、分层解耦、删除/重命名前先确认

## 1. 项目目标与边界

### 1.1 目标
- 构建单页面旋转台控制软件（Qt Widgets）
- 打通板卡通信、运动控制、粗/精回零、相机、光机同步投影、安全联锁、日志与配置
- 达到技术文档首版验收标准

### 1.2 边界（本期不做）
- 多页面 UI 架构
- Z 轴电动控制（仅手动到位确认）
- 与当前里程碑无关的大规模重构

## 2. 架构与模块责任
- `MainWindowController`：UI 事件编排与状态展示
- `BoardAdapter`：`COM_GAS_N` API 封装、返回码统一、超时机制
- `MotionService`：JOG、绝对/相对运动、角度脉冲换算
- `HomingService`：粗零点自检、精零点边沿中点法
- `SafetyManager`：联锁策略、异常停机、状态切换到 `ALARM`
- `CameraService`：枚举/预览/拍照/MP4录像/参数调整
- `ProjectorService`：光机命令发送（`WT+PWRE/WT+LEDE/WT+LEDS`）与状态回读
- `ImageSequenceService`：图片目录导入、角度编号解析、序列校验
- `SyncPlaybackService`：旋转与投影同步、循环次数/无限循环控制
- `ConfigManager`：参数读写与版本化默认值
- `LogService`：操作日志、告警日志、追溯字段

## 3. 里程碑总览（M1~M7）

### M1 项目骨架
- 目标：建立可运行的单页面框架 + 日志 + 配置
- 输出：可启动主窗口、基础控件占位、配置文件可读写、日志可落盘
- 验收：应用可启动，配置项生效，日志含时间/级别/模块

### M2 板卡接入
- 目标：打通连接、复位、断开与 DI 实时显示
- 输出：`BoardAdapter` 初版 + DI 轮询线程
- 验收：可稳定连接板卡；DI0~DI3 在 UI 刷新；错误码可见

### M3 基础运动
- 目标：实现 JOG、绝对/相对运动与停止
- 输出：`MotionService` 初版 + 角度脉冲换算
- 验收：`1 deg = 250 pulse` 计算一致；急停可立即生效

### M4 粗零点自检
- 目标：上电强制粗零点流程与联锁
- 输出：`SELF_CHECK_COARSE` 状态流程
- 验收：未通过粗零点前禁用运动按钮；失败进入 `ALARM`

### M5 精零点
- 目标：边沿采集 `P1/P2`，计算 `P0=(P1+P2)/2`
- 输出：`FINE_HOMING` 闭环（成功/失败）
- 验收：重复执行可追溯；超时/单边沿触发告警

### M6 相机与光机模块
- 目标：UVC 枚举、预览、拍照、MP4 录像、参数调节；光机开关与亮度接入
- 输出：`CameraService` + `ProjectorService` 与 UI 对接
- 验收：相机稳定预览与录像；光机开关/LED开关可控；亮度百分比可用

### M7 安全与验收
- 目标：完成旋转与投影同步循环、完善联锁、日志、参数固化与联调测试
- 输出：验收测试记录与问题清单闭环
- 验收：满足文档第14节全部条目

## 4. 每个里程碑的执行模板（固定）
1. 对齐目标：仅声明本次覆盖的里程碑与验收点
2. 影响文件：先列出拟修改文件（不跨模块泛改）
3. 最小实现：只做本闭环必需改动
4. 本地验证：编译/运行/关键流程自检
5. 结果记录：改动摘要 + 风险 + 下一步最小任务

## 5. 分步任务分解（可直接执行）

### M1 任务清单
1. 创建项目目录骨架（`src/`, `include/`, `config/`, `logs/`）
2. 建立主窗口与单页面布局骨架（设备区/运动区/回零区/Z确认区/相机区/光机同步区/日志区）
3. 完成 UI 文案中文化基线（按钮/标签/状态/告警）
4. 确认项目文本文件统一 UTF-8 编码（源码/配置/文档）
5. 实现 `ConfigManager`（默认参数、读写、校验）
6. 实现 `LogService`（文件+UI输出通道）
7. 打通 UI 启动路径，接入配置和日志

### M2 任务清单
1. 定义 `BoardAdapter` 接口与错误码映射
2. 实现连接/复位/断开
3. 实现 DI 轮询线程与去抖配置入口
4. 将 DI 状态绑定到 `diIndicator[0..3]`
5. 加入断线重试与告警日志

### M3 任务清单
1. 实现角度<->脉冲转换器
2. 实现 `JOG+/JOG-/STOP/ESTOP`
3. 实现绝对/相对运动指令链路
4. 状态切换：`READY/JOGGING/POSITIONING`
5. 运动结果与错误上报到日志

### M4 任务清单
1. 开机后强制进入 `SELF_CHECK_COARSE`
2. 实现 DI1 触发等待 + 超时
3. 实现退回脉冲 + 二次慢速接近
4. 自检通过写入粗零参考并转 `READY`
5. 自检失败进入 `ALARM` 并锁定运动

### M5 任务清单
1. 实现低速扫描与边沿采集
2. 记录 `P1/P2` 并计算 `P0`
3. `GA_ZeroPos` 或零偏设置流程
4. 异常分支：单边沿/超时
5. 结果追溯字段写入日志

### M6 任务清单
1. 枚举相机并支持切换
2. 预览与拍照
3. MP4 录像开始/停止
4. 参数滑条与模式按钮接入
5. 光机串口枚举与连接管理（USB 动态 COM）
6. 光机总开关/LED开关/亮度百分比接入（发送协议命令）
7. 录像与拍照错误处理和日志

### M7 任务清单
1. 图片目录导入、角度序列解析与校验
2. 同步编排：连续旋转并按角度触发投影对应图片
3. 循环控制：有限循环次数与无限循环
4. 完整联锁矩阵（通信丢失/急停/报警）并覆盖同步流程
5. 参数固化与启动加载一致性
6. 关键流程回归测试脚本/清单
7. 重复性测试与偏差记录
8. 验收报告与遗留项

## 6. 横向工程规范
- 每次只实现一个最小闭环，不并行推进多个里程碑
- 新增逻辑必须放在对应模块，禁止 UI 直调板卡 DLL
- 每条板卡动作必须做：返回码检查 + 超时 + 日志
- DI 极性、去抖、稳定时间必须配置化
- 操作员可见 UI 文案统一中文，不混用英文按钮/状态词
- 项目文本文件统一 UTF-8 编码（建议无 BOM），避免中文乱码
- 涉及删除/重命名/批量替换时，先提交清单并等待确认

## 7. 风险与前置确认
- 现场确认项（来自技术文档第15节）应在 M4 前闭环关键参数
- 若现场 DI 极性与默认值不一致，需要优先完成可配置极性验证
- MP4 编码兼容性需在目标机提前验证（M6 前半段）
- 光机串口动态 COM 识别与重连策略需在目标机优先验证（M6 前半段）
- 光机命令结束符（Enter 对应 CR/CRLF）需在目标机优先验证（M6 前半段）
- 图片文件命名规范与角度步距一致性需要在联调前冻结（M7 前半段）

## 8. 立即执行策略（从现在开始）
- 第一步：执行 `M1-任务1`，只创建最小项目目录与占位文件
- 第二步：执行 `M1-任务2`，落主窗口单页面骨架
- 第三步：执行 `M1-任务3`，接入 `ConfigManager` 最小可用版
- 新增需求落位：M6（光机接入）+ M7（旋转投影同步循环）
- 每一步完成后都先给你确认，再进入下一步
