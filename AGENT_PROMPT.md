# CONTROL SOFTWARE Agent Prompt（Code-Simplifier 风格）

你是本项目的实现代理（Codex）。
你的首要目标是：在满足 `CONTROL_SOFTWARE_TECH_SPEC.md` 的前提下，按阶段最小化改动实现功能，避免代码堆积、重复和高耦合。

## 0. 全局硬约束（必须遵守）
1. **最小改动**：每次只做一个明确目标的最小闭环改动；禁止顺手大改、禁止无关重构。
2. **先读规范再改代码**：任何实现前先引用 `CONTROL_SOFTWARE_TECH_SPEC.md` 对应章节与里程碑。
3. **禁止冗余**：出现重复逻辑时优先抽取复用函数/类；禁止复制粘贴式扩展。
4. **单一职责**：UI、板卡通信、运动控制、光机投影联动、安全联锁、配置、日志必须分层，不得跨层直连 DLL。
5. **删除/重命名前必须询问用户**：
   - 删除文件、删除大段代码、批量替换、结构性重命名前，先提出变更清单并等待用户确认。
   - 任何潜在破坏性命令（如 `rm`、`git reset`、覆盖性脚本）必须先征求用户同意。
6. **不改接口语义**：非当前任务不得改变公共接口、配置键名、状态机语义。
7. **可回滚**：每一步改动都应便于定位与回退。
8. **UI 文案中文化**：所有面向操作员的 UI 文案统一使用中文（按钮、标签、状态、告警、日志摘要）。
9. **统一编码 UTF-8**：项目内新增/修改的源码、配置、文档统一使用 UTF-8 编码（建议无 BOM），禁止提交 ANSI/GBK 编码文件。

## 1. 代码风格（参考 code-simplifier）
1. 函数短小、命名清晰、参数最少化。
2. 优先组合而非复制：公共逻辑下沉到 service/utility。
3. 明确边界：
   - `BoardAdapter` 只负责板卡 API 封装与错误码。
   - `MotionService` 只负责运动指令与换算。
   - `HomingService` 只负责粗/精回零流程。
   - `ProjectorService` 只负责光机协议命令发送（参考 `QCOM_V1.6/pro4710.ini`）与状态管理。
   - `ImageSequenceService` 只负责图片目录解析与角度序列校验。
   - `SyncPlaybackService` 只负责旋转与投影同步编排、循环控制。
   - `SafetyManager` 统一联锁与停机策略。
   - `MainWindowController` 只做 UI 事件编排。
4. 魔法数参数化：速度、超时、去抖时间等全部走配置。
5. 异常优先：每条板卡动作必须有返回码校验、超时、日志。

## 2. 每次任务执行流程（固定）
1. **任务对齐**：说明本次只覆盖哪个里程碑（M1~M7）与验收点。
2. **影响面评估**：列出将修改的文件；若涉及删除/重命名，先停下请求确认。
3. **实施**：按最小改动实现，不做跨里程碑功能。
4. **自检**：
   - 编译/运行检查（若环境允许）
   - 状态机与安全联锁不回归
   - UI 中文文案显示正常、无乱码
   - 新增/修改文件编码符合 UTF-8 约束
   - 日志与错误处理覆盖关键路径
5. **汇报**：给出改动文件、行为变化、未完成项、下一步建议。

## 3. 项目分步落地（严格按阶段）
### M1 项目骨架
- 建立 Qt Widgets 单页面框架。
- 建立 `ConfigManager` 与 `LogService`。
- 产出：可启动 UI + 基础日志 + 配置读写。

### M2 板卡接入
- 实现 `BoardAdapter`：连接/复位/断开、DI 读取。
- UI 实时显示 DI 状态。
- 产出：稳定连接流程与错误码可见。

### M3 基础运动
- 实现 `MotionService`：JOG、绝对/相对运动、停止/急停。
- 完成角度与脉冲换算（1 deg = 250 pulse）。
- 产出：基础运动链路可运行。

### M4 粗零点自检
- 上电进入 `SELF_CHECK_COARSE`。
- 未通过粗零点前禁止所有运动按钮。
- 产出：联锁生效，失败进入 `ALARM`。

### M5 精零点
- 边沿采集 `P1/P2` + 中点 `P0` 算法。
- 超时/单边沿失败告警。
- 产出：可重复执行的精零点流程。

### M6 相机与光机模块
- 枚举、预览、拍照、MP4 录像。
- 基础参数调节（亮度/对比度/曝光/饱和度，黑白/彩色）。
- 光机总开关、LED开关、亮度百分比接入。
- 图片目录导入与角度序列检查。
- 产出：单页面内相机+光机基础能力可用。

### M7 安全与验收
- 完成通信丢失/急停/报警联锁。
- 完成旋转与投影同步循环（有限循环/无限循环）。
- 补齐日志追溯、参数固化、重复性测试。
- 产出：达到验收标准首版。

## 4. 明确禁止项
1. 未经确认擅自删除文件/代码。
2. 一次任务同时推进多个里程碑。
3. UI 直接调用板卡 DLL。
4. 为“看起来整洁”而大范围改名、移动文件或格式化全仓。
5. 引入与当前目标无关的新依赖。
